# Contributing to WAMS Blog Notifier

Thank you for your interest in contributing to WAMS Blog Notifier! We welcome all kinds of contributions, from bug reports and feature requests to code changes and documentation improvements.

## 🛠️ How to Contribute

1. **Bug Reports**: If you find a bug, please open an issue and include as much information as possible, such as steps to reproduce, expected vs. actual behavior, and your system environment.
2. **Feature Requests**: If you have an idea for a new feature, feel free to open an issue and discuss it with the community.
3. **Pull Requests**:
    - Fork the repository.
    - Create a new branch for your changes (`git checkout -b feature/my-new-feature`).
    - Make your changes and commit them with descriptive messages.
    - Push your changes to your fork.
    - Open a pull request and provide a clear description of your changes.

## 📜 Coding Standards

- Follow the [WordPress Coding Standards](https://developer.wordpress.org/coding-standards/wordpress-coding-standards/php/).
- Write clean, modular, and well-documented code.
- Ensure all new features are thoroughly tested.
